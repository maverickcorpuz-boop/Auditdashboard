☁ Enable Shared Cloud Data
Follow all 5 steps so every user sees the same data
⚠ Right now data is saved only on YOUR browser. Your colleagues see empty screens because each browser has its own private localStorage. Complete the steps below — it takes about 5 minutes and is completely free.
1
Create a free Supabase account
→ Open https://supabase.com in a new tab
→ Click Start your project → Sign up with GitHub or email
→ Click New Project → Name it (e.g. mc-audit) → Set any DB password → Create project
→ Wait about 2 minutes for it to be ready ✅
2
Create the database table
→ In your Supabase project, click SQL Editor in the left sidebar
→ Click New query
→ Paste the SQL below and click ▶ Run
create table if not exists iad_appstate ( id text primary key, value text not null, updated_at timestamptz default now() ); alter table iad_appstate enable row level security; create policy "allow all" on iad_appstate for all using (true) with check (true);
3
📋 Copy your API credentials — HERE IS HOW:
→ In Supabase, look at the left sidebar
→ Click the ⚙ gear icon at the very bottom left → click Settings
→ In the Settings menu, click API (under "Configuration")
→ You will see two things to copy:
① Project URL
Under "Project URL" heading
https://abcdefgh.supabase.co
Click the copy icon next to it
② Anon Public Key
Under "Project API keys" → anon row
eyJhbGciOiJIUzI1NiIs...
Click the copy icon on that row
⚠ IMPORTANT: The anon key must start with eyJ... — if you see sb_publishable_... that is the wrong key and will not work. Look for the JWT / anon key in the API settings page.
⚠ Copy the anon key only — NOT the service_role key
🔧 Test your connection
4
Paste credentials into index.html
→ Open index.html in Notepad, VS Code, or any text editor
→ Press Ctrl+F (Find) and search for: YOUR_PROJECT_ID
→ Replace both lines below with your copied values:
const SUPA_URL = 'https://YOUR_PROJECT_ID.supabase.co'; ← replace with ① Project URL
const SUPA_KEY = 'YOUR_ANON_PUBLIC_KEY'; ← replace with ② Anon key
→ Save the file → Go to GitHub → upload/replace index.html → wait for GitHub Pages to update (~1 min)
5
Push your existing data to the cloud
→ Open the app on GitHub Pages → Log in as Admin
→ Click the badge in the top bar → Click the button below
→ This uploads all your existing records so colleagues see them
✅ After setup: Everyone opens the same GitHub Pages URL → logs in → sees the same shared data. Any change one user makes is instantly visible to all others.
